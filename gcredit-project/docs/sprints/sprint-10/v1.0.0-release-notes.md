- User registration, login, logout, password reset

## Security

- **Authentication:** JWT dual-token (access 15min + refresh 7d with rotation)
- **Authorization:** RBAC with 4 roles, route-level guards
- **Headers:** Helmet CSP, CORS whitelist
- **Rate Limiting:** 60 req/min global, configurable per-endpoint
- **IDOR Protection:** Ownership checks on all resource endpoints
- **Transaction Safety:** Prisma `$transaction` with ReadCommitted isolation
- **Input Sanitization:** CSV injection (`sanitizeCsvField`), XSS (`sanitize-html`)
- **Password Security:** bcrypt hashing, strength validation
- **File Upload:** Multer with 100KB limit, type validation
- **Azure Storage:** SAS tokens with 5-min expiry for evidence files

## Deployment Instructions

### Prerequisites
- Node.js 20.20.0 LTS or higher
- npm 10+
- Azure PostgreSQL Flexible Server
- Azure Blob Storage account
- Microsoft 365 tenant (for Graph API email)

### Setup

```bash
# 1. Clone and checkout
git clone https://github.com/legendyz/G-Credit.git
cd G-Credit/gcredit-project

# 2. Backend setup
cd backend
npm install
cp .env.example .env
# Edit .env with your Azure credentials
npx prisma migrate deploy
npm run build
npm run start:prod

# 3. Frontend setup
cd ../frontend
npm install
npm run build
# Serve dist/ with any static file server

# 4. Verify
curl http://localhost:3000/health
# â†’ {"status":"ok"}
```

### Environment Variables

See `backend/.env.example` for the complete list. Key variables:
- `DATABASE_URL` â€” Azure PostgreSQL connection string
- `AZURE_STORAGE_CONNECTION_STRING` â€” Blob Storage connection string

- `MS_GRAPH_*` â€” Microsoft Graph API credentials for email
